# Security Policy

## Authorized Use Only

Neap is a penetration testing tool designed **exclusively for authorized security assessments**. You must obtain explicit, written permission from the system owner before using this tool against any target. Unauthorized access to computer systems is illegal under the Computer Fraud and Abuse Act (CFAA) and equivalent laws worldwide.

## Responsible Use Policy

- **Always** obtain written authorization before testing any system you do not own.
- **Limit** your testing to the scope defined in your authorization agreement.
- **Document** all activities during an engagement.
- **Report** all findings to the system owner promptly and confidentially.
- **Never** use this tool for malicious purposes, unauthorized access, or any activity that violates applicable laws.

## Reporting Security Vulnerabilities

If you discover a security vulnerability in Neap itself, please report it responsibly:

1. **Email:** Send details to the project maintainers via the contact information in the repository.
2. **GitHub Issues:** For non-sensitive issues, open a GitHub Issue with the `security` label.

Please include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge receipt within 48 hours and aim to provide a fix or mitigation plan promptly.
